# 终端集成与无框方案（Windows Terminal & Deco-Terminal）

## 结论摘要
- 纯 CLI/TUI 应用无法直接去除系统窗口边框。
- 在 Windows 上可以用 Windows Terminal 的 Focus Mode（隐藏标题栏和标签栏）配合 Fullscreen/Maximized，获得接近“无框”的效果。
- 已加入 Windows Terminal 自动启动逻辑，默认关闭，可在 `settings.json5` 中启用。
- 初步提供 Deco-Terminal 模式：对传统控制台窗口进行“无边框”样式调整，作为自研终端外壳的起点。

## 启用方式（Windows Terminal）
在用户配置 `settings.json5` 中加入/修改：

```json5
{
    "terminal_integration": {
        "enabled": true,
        "backend": "windows_terminal",
        "fallback_backend": "classic",
        "prefer_bundled_wt": true,
        "bundled_wt_path": "",
        "use_wt_portable_mode": true,
        "bundled_wt_auto_setup_profile": true,
        "bundled_wt_profile_name": "DecoScreenBeautifier-CRT",
        "bundled_wt_settings_path": "",
        "bundled_wt_retro_effect": true,
        "bundled_wt_enable_pixel_shader": false,
        "bundled_wt_pixel_shader_path": "",
        "bundled_wt_color_scheme": "Campbell",
        "bundled_wt_use_acrylic": true,
        "bundled_wt_opacity": 88,
        "bundled_wt_font_face": "Cascadia Mono",
        "bundled_wt_font_size": 14,
        "bundled_wt_enable_focus_toggle_binding": true,
        "focus_mode_toggle_key": "alt+shift+f",
        "focus_mode": true,
        "fullscreen": false,
        "maximized": true,
        "window_target": "new",
        "profile": "",
        "title": "DecoScreenBeautifier",
        "starting_directory": "",
        "position": "",
        "size": "",
        "deco_borderless": true,
        "deco_topmost": false,
        "deco_position": "",
        "deco_size": ""
    }
}
```

建议：
- `prefer_bundled_wt=true` 时会优先查找项目目录下内置 WT（默认查找 `vendor/windows_terminal/x64/WindowsTerminal.exe`）。
- `bundled_wt_path` 可指定自定义内置 WT 路径（可填目录或 `WindowsTerminal.exe` 文件路径）。
- `use_wt_portable_mode=true` 时会为内置 WT 自动创建 `.portable` 标记，隔离配置与状态，避免影响用户全局 WT 设置。
- `bundled_wt_auto_setup_profile=true` 时，启动前会自动在便携配置中创建/更新 `DecoScreenBeautifier-CRT` Profile。
- `bundled_wt_retro_effect=true` 时会写入 `experimental.retroTerminalEffect`（并兼容写入 `retroTerminalEffect`）。
- `bundled_wt_enable_pixel_shader=true` 时会写入 `experimental.pixelShaderPath`（并兼容写入 `pixelShaderPath`），默认占位 shader 为 `vendor/windows_terminal/shaders/deco_placeholder.hlsl`。
- `bundled_wt_enable_focus_toggle_binding=true` 时会自动写入 `toggleFocusMode` 快捷键（默认 `alt+shift+f`）。
- 运行中可通过应用底部按钮 `Toggle Border (B)` 或按 `B` 键触发切换，便于“先拖动窗口，再恢复无框”。
- `bundled_wt_settings_path` 可手动指定便携 settings 文件（默认 `vendor/windows_terminal/x64/settings/settings.json`）。
- 想要更接近“无框”，可以把 `fullscreen` 设为 `true`。
- `profile` 可以填写你在 Windows Terminal 中的自定义配置名称（含复古效果、字体等）。
- `position`/`size` 对应 Windows Terminal 的 `--pos` 与 `--size` 参数（格式为 `x,y` 与 `cols,rows`）。

## 内置 WT 的 Profile 自动初始化

当命中“内置 WT”时，启动器会自动执行一次 profile 初始化：

- 目标 profile：`terminal_integration.bundled_wt_profile_name`（默认 `DecoScreenBeautifier-CRT`）
- 默认写入项：字体（Cascadia Mono 14）、Acrylic、透明度、`experimental.retroTerminalEffect`
- 可选写入项：`experimental.pixelShaderPath`（启用 `bundled_wt_enable_pixel_shader` 后）
- 配置文件位置：默认 `WindowsTerminal.exe` 同级目录下 `settings/settings.json`

该初始化仅针对内置 WT 的 Portable 配置目录，不会写入系统 WT 的用户目录。

## 启用方式（Deco-Terminal 初始实现）
将 `backend` 设为 `deco_terminal`：

```json5
{
    "terminal_integration": {
        "enabled": true,
        "backend": "deco_terminal",
        "fallback_backend": "classic",
        "deco_borderless": true,
        "deco_topmost": false,
        "deco_position": "",
        "deco_size": "",
        "deco_auto_fit": true,
        "deco_monitor": "auto",
        "deco_use_work_area": false,
        "deco_effects": {
            "enabled": true,
            "scanlines": { "enabled": true, "spacing": 2, "intensity": 0.15, "speed": 1, "mode": "darken" },
            "glow": { "enabled": true, "intensity": 0.35, "radius": 1, "halo_alpha": 0.35 },
            "noise": { "enabled": true, "density": 0.02, "chars": " .`,", "color": "", "on_text": false },
            "warp": { "enabled": true, "strength": 1, "probability": 0.12 }
        }
    }
}
```

说明：
- 该模式仅对传统控制台窗口（conhost）生效，无法作用于 Windows Terminal。
- 若系统默认终端为 Windows Terminal，程序会尝试对当前终端窗口去边框，但仍可能受系统限制影响。
- `deco_position` / `deco_size` 为像素坐标与像素尺寸，留空或设为 `auto` 会走自动适配。
- `deco_auto_fit` 为 `true` 时，缺失的 `deco_position`/`deco_size` 将自动填充。
- `deco_monitor` 可选值：`auto`（优先副屏/非主屏）、`primary`、`secondary`、或 0 开始的屏幕索引。
- `deco_use_work_area` 为 `true` 时使用工作区（避开任务栏），否则铺满整个屏幕区域。
- `deco_effects` 用于配置复古特效管线，支持扫描线、辉光、噪点与轻微畸变等参数。
- 当 Deco-Terminal 处理失败时，会按 `fallback_backend` 退回（例如 `classic` 或 `windows_terminal`）。

## Windows Terminal 推荐设置（可选）
在 Windows Terminal 的 `settings.json` 中，为目标 profile 添加复古特效与透明度，例如：

```json5
{
    "retroTerminalEffect": true,
    "useAcrylic": true,
    "opacity": 0.9,
    "font": { "face": "Cascadia Mono", "size": 14 }
}
```

## 回退与禁用
- 启动时会按顺序查找：内置 WT（可配置） -> 系统 `wt.exe`。
- 若内置缺失但系统 WT 可用，会输出明确回退日志并继续通过系统 `wt.exe` 启动。
- 若内置与系统 WT 均不可用，将自动回退到当前终端，并输出“已检查路径 + 补齐指引”日志。
- 临时禁用可设置环境变量：`DSB_DISABLE_WT=1`。
