# Deco-Terminal 开发待办与备忘

目标：提供一套“自研终端外壳（Deco-Terminal）”，与核心 TUI 逻辑解耦，支持无边框与复古风格；同时保留经典终端回退方案。

## 现状
- 已支持终端后端切换：`classic` / `windows_terminal` / `deco_terminal`
- 已实现 Deco-Terminal 的无边框基础处理（仅限 conhost）
- 已支持失败回退（按 `fallback_backend`）

## 设计原则
- **解耦**：核心 TUI 功能不依赖具体终端宿主。
- **回退**：任一自研能力失效时可回到系统终端。
- **渐进**：先构建宿主壳与效果管线，再逐步完善渲染能力。

## 关键模块拆分（拟）
- `terminal_launcher`: 负责后端选择与启动策略
- `deco_terminal`: 负责窗口壳与特效管线（无边框/扫描线/辉光）
- `renderer_core`（待建）：输出标准化的字符网格与样式信息
- `host_adapters`（待建）：不同宿主的适配层（经典终端 / Windows Terminal / Deco）
