# 观察但未处理的问题 (Observed but Unresolved Issues)

本文档用于记录在开发过程中发现的、但尚未处理或优化的潜在问题。

## 🧩 组件与 UI (Components & UI)
- [ ] **音频可视化性能**：在 `update_interval` 较低（如 0.03s）时，频繁的 FFT 计算和 UI 重绘可能会导致 CPU 占用升高，尤其是在低性能副屏设备上。
- [ ] **GPU 监控缺失**：目前 `HardwareMonitor` 仅支持 CPU 和内存，尚未集成 GPU 使用率监控（需要 `pynvml` 或 `GPUtil`，且涉及不同驱动的兼容性）。
- [ ] **CPU 核心数显示限制**：`HardwareMonitor` 组件中硬编码了最多显示 8 个 CPU 核心 (`if i >= 8: break`)。对于拥有更多核心的现代 CPU（如 12核、16核等），这会导致无法显示所有核心的状态。建议实现分页显示或动态网格布局。
- [ ] **字体适配**：Textual 默认不支持在大尺寸组件中使用 ASCII 艺术字体，目前只能通过普通文本模拟，视觉冲击力有待提升。
- [ ] **模板布局适配不足**：当前模板通过 CSS 类定义布局，无法基于模板数据动态重排组件，用户自定义布局与模板切换尚未融合。
- [ ] **编辑器拖拽/缩放未实装**：目前主要通过属性面板输入数值调整位置和大小，仍需补齐拖拽与拉伸交互。
- [ ] **GUI 版编辑器能力仍有限**：已实现网格拖拽与尺寸调整（方案 B V2 首版），但尚未支持多选、对齐参考线、撤销/重做与组件工具箱新增。
- [ ] **全局缩放范围有限**：`global_scale` 目前主要作用于图像组件，GIF/其他组件的像素化与密度控制仍需统一适配。

## 🎞️ GUI 后处理 (CRT Shader)
- [ ] **CRT 后处理当前为 CPU 路线**：当前 `QImage + NumPy` 后处理已可用，但在高分辨率/高 FPS 场景下存在性能压力；后续需评估 Qt Quick / OpenGL 路线。
- [~] **环境依赖敏感**：部分机器上 `PySide6` 运行可能出现 Qt DLL 缺失导致 GUI 无法启动；已在 `src/main_gui.py` 增加 Windows Qt DLL 路径预热（`PATH` + `os.add_dll_directory`）、启动前 PATH 去 conda 污染、失败日志 `qt_runtime_error.log`、DLL 逐项探测/预加载记录与 VC++ 官方下载提示；并在 `DecoScreenBeautifier.spec` 中全面排除了 Anaconda 来源 DLL（CRT/UCRT/ICU/API-SET），统一走 PySide6 侧运行库；仍需在更多目标机器实测（尤其是未安装 VC++ 运行库的纯净系统）。
- [ ] **当前默认后处理性能压力较高**：2026-02-07 本机基准（`scripts/benchmark_gui_host.py`）在开启 `effects+crt_shader` 时，1080x480 平均约 12.31 FPS / 98.36% CPU，1920x480 平均约 7.32 FPS / 99.37% CPU，需继续优化后处理与绘制路径。

## 🛠️ 系统与稳定性 (System & Stability)
- [ ] **音频设备独占**：在某些 Windows 系统上，`pyaudio` 可能会因为设备被其他程序独占而初始化失败，目前的 Mock 模式只是临时方案。
- [x] **多显示器识别**：已支持自动检测并锁定到副屏的窗口定位。
- [~] **外部终端分发策略待持续验证**：已于 2026-02-07 完成增强包实打包与 PoC 验证（内置 WT + Portable profile 初始化可用），但仍需在更多“无系统 WT”目标机器回归启动稳定性、签名发布策略与许可证告知流程。
- [~] **内置 WT Profile 字段兼容风险**：已在 `src/utils/terminal_launcher.py` 同时写入 `experimental.retroTerminalEffect` / `retroTerminalEffect` 与 `experimental.pixelShaderPath` / `pixelShaderPath` 以适配不同版本，但后续仍需在更多 WT 版本做回归，避免配置 schema 变动导致字段失效。
- [ ] **解释器混用风险**：当 conda/base 与 venv 同时激活时，`python` 可能指向非 venv，导致依赖未安装而启动失败；当前仅有启动提示，建议提供专用启动脚本或在文档中强化说明。
- [ ] **Qt 运行时依赖缺失会阻塞 GUI 验收**：`scripts/validate_gui_host.py` 与 `scripts/benchmark_gui_host.py` 在 2026-02-07 本机执行时均报 `DLL load failed while importing QtCore`；需安装 Microsoft VC++ 2015-2022 x64 运行库后复测，并考虑在发布安装包中增加前置检测。
- [x] **Qt 运行时依赖阻塞（本机）已解除**：2026-02-07 将项目 `venv` 迁移为 Python 3.11 后，`PySide6.QtCore/QtGui/QtWidgets` 导入正常，功能验收脚本可通过。
- [~] **legacy 终端 EXE 启动链路易误导**：当 `terminal_integration.enabled=true` 且 `backend=windows_terminal` 时，旧版 `DecoScreenBeautifier_legacy_terminal.exe` 会先拉起 Windows Terminal 再自行退出，容易被误判为“启动失败且无弹窗”；2026-02-07 已在 `src/main.py` 对 legacy 冻结入口禁用该二次拉起，并补充统一异常弹窗与 `legacy_terminal_error.log`，仍需在更多终端环境回归验证。
