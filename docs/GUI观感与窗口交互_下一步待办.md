# GUI 观感与窗口交互：下一步待办（2026-02-07）

> 背景：当前 GUI 版本已可运行，但存在明显体验问题：
> 1) 文字排布混乱；2) 画面持续抖动；3) 强制全屏；4) 无法调节窗口大小。

## 目标与验收标准

- [ ] **视觉稳定**：常规运行 10 分钟内，文本位置无肉眼可见的持续跳动。
- [ ] **排版可控**：同一模板在固定窗口尺寸下，多次启动布局一致。
- [ ] **窗口可控**：支持窗口化运行，允许拖拽移动与手动缩放。
- [ ] **全屏可选**：全屏/无边框仅作为可切换选项，不再强制。

## A. 文字排布混乱（高优先级）

- [ ] **统一字符网格基线**
  - 统一 `font_face / font_size / cell_aspect` 的生效链路（配置 -> 渲染上下文 -> 组件）。
  - 明确“逻辑列宽/行高”与“像素尺寸”的映射规则，避免组件各自换算。
- [ ] **收敛布局重算触发条件**
  - 将“数据刷新”和“布局重排”拆分：数据变更只触发内容更新，不触发全量布局。
  - 仅在窗口尺寸、模板切换、全局缩放变化时重建网格。
- [ ] **组件最小尺寸与溢出策略**
  - 为关键组件定义最小宽高，避免在窄区域内出现文本覆盖。
  - 为超长文本增加截断/省略策略，避免挤压邻近区域。

## B. 画面抖动/跳动（高优先级）

- [ ] **降低全屏重绘频率**
  - 优先更新脏区域（dirty rect），避免每帧全屏拷贝。
  - 限制高频组件刷新节奏（时钟、监控曲线）并统一帧节拍。
- [ ] **解耦采样频率与渲染频率**
  - 数据采样可高频，渲染输出按稳定 FPS（如 30/45/60）节流。
  - 引入“最大每秒重排次数”阈值，防止布局抖动。
- [ ] **后处理抖动治理**
  - 对 scanline/noise/warp 的随机参数增加帧间平滑，避免突变。
  - 默认配置下限制过强特效，优先保证可读性。

## C. 强制全屏与窗口不可调（高优先级）

- [ ] **窗口模式开关落地**
  - 在 `gui_host` 配置补齐并验证：`borderless`、`use_work_area`、`size_px`、`pos_px`。
  - 提供默认“窗口化”配置，保留“无边框全屏”作为可选项。
- [ ] **窗口缩放能力**
  - 允许在 windowed 模式下使用系统边框与尺寸拖拽。
  - 监听 resize 事件后仅触发必要重排，避免拖动时严重卡顿。
- [ ] **窗口状态持久化**
  - 记录并恢复：窗口模式、位置、尺寸、置顶状态。
  - 多显示器场景下做越界修正（显示器变更后自动回退到可见区域）。

## D. 可观测性与验证

- [ ] **新增运行指标**
  - 记录 FPS、重绘面积比例、布局重排次数、组件刷新耗时。
  - 输出到 `performance_monitor` 日志，便于对比优化前后。
- [ ] **回归验收脚本增强**
  - 扩展 `scripts/benchmark_gui_host.py`：增加“窗口化 + 可缩放”测试场景。
  - 增加“静态模板稳定性”检查（连续 N 帧布局哈希一致）。

## 实施顺序建议

- [ ] **第 1 批（先稳）**：A2 / B1 / C1
- [ ] **第 2 批（再顺）**：A1 / B2 / C2
- [ ] **第 3 批（再美）**：A3 / B3 / C3 / D
