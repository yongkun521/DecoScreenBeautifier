# 自定义界面说明

本文档说明当前主线版本（`dist/DecoScreenBeautifier.exe`，内置 Windows Terminal 宿主）的主要自定义方式。

## 1. 快速入口

- 主界面底部 Footer 里有 `Zoom -` / `Zoom +` 两个快捷项。
  - 作用等价于 Windows Terminal 里的 `Ctrl+-` / `Ctrl++`。
  - 可以直接点击，不需要记快捷键。
  - 当前内部实现会给内置 Windows Terminal 自动写入一组专用缩放热键，以避免不同键盘布局下 `Zoom +` 失效。
- `T`：打开模板选择器。
- `E`：打开布局编辑器。
- `B`：切换是否显示边框（本质是切换 WT Focus Mode）。
- `H`：切换标题栏。
- `M`：切换底部工具栏。

## 2. 模板与主题

现在除了原来的 `Redline 2077` 之外，还新增了两套偏“轻框图形”方向的模板：

- `Signal Lattice Max`
  - 风格：超宽轻框信息墙，主视觉图像 + 遥测轨道 + 信号条带。
  - 适合分辨率：`1920x480` 一类的超宽副屏。
  - 图片默认：`Pixel + Fill and Crop`
- `Deck Strip V2`
  - 风格：轻框状态条，重点突出 `Ticker / Clock / Network / Badge`。
  - 适合分辨率：`1920x360` 一类的矮条屏。
- `Redline 2077`
  - 风格：偏红的 2077 赛博面板。
  - 主题类：`theme-redline-2077`
  - 字符预设：`edgerunner_block`
  - 适合分辨率：`1920x480` 一类的超宽副屏。

建议先按 `T` 试模板，再决定要不要继续改布局。很多“观感不对”或者“空间不够”的情况，本质上是当前模板的网格和组件变体不匹配。

## 3. 布局编辑器怎么用

当前 Textual 版编辑器是“网格编辑器”，不是自由像素拖拽。它保存的是每个组件在网格里的：

- `Column`
- `Row`
- `Column Span`
- `Row Span`

使用步骤：

1. 按 `E` 打开编辑器。
2. 左侧 `Components` 里选一个组件类型。
3. 点 `Add Component` 添加到当前模板布局。
4. 中间 `Layout Preview` 会显示一个字符网格预览。
5. 如果要删除组件，可以先在中间 `Components` 列表里选中它，再点 `Delete Selected`。
6. 如果你想保留一行空白，可以点右侧网格区的 `Add Row`；不想要了就点 `Remove Row`。
7. 右侧 `Properties` 里改位置和占用格数。
8. 右侧还可以改当前组件的 `Visual Variant`。
9. 如果是图片组件，还可以改 `Image Display` 和 `Image Render`。
10. 组件属性现在会在输入有效后自动应用到当前预览，不需要再点 `Apply Changes`。
11. 点 `Save Layout` 才会把当前布局真正写入布局文件。
12. 如果你直接按 `Esc` 离开而忘了保存，编辑器会弹出提醒。
13. 按 `Esc` 返回主界面。

现在主显示界面已经会读取你保存的 `layout.json5`，所以保存并返回后，主界面会按你的布局重新排。
主界面的网格行列数现在也会跟着布局文件一起变，所以如果你删掉底部空白行，返回主界面后那一行也会一起消失。

默认行为补充：

- 删除组件后，完全空白的行会自动收缩。
- 完全空白的列也会自动收缩，所以如果某一排本来最多放 3 块、现在只剩 2 块，而且第三列全局也没人占用，就会自动适应宽度。
- 如果你是故意想保留一行空白，请用 `Add Row` 显式添加，这样系统不会自动把它吃掉。
- 右侧 `Properties` 里的组件属性会自动应用到当前布局预览，但这不等于已经保存到磁盘。
- `Save Layout` 的职责现在更明确：它只负责把当前布局写入 `layouts/<template_id>.json5`。
- 如果窗口比较窄或比较矮，编辑器和模板选择器会自动切成更紧凑的排版，底部操作按钮也会尽量保持可达。

## 4. 为什么会提示“空间不足”

这通常不是报错，而是当前模板网格里没有足够大的连续空位。

比如：

- 你要加一个默认占 `4x2` 的硬件面板。
- 但当前模板只剩下零散空格，没有连续的 `4x2` 区域。

这时可以这样处理：

1. 先缩小一个已有组件的 `Span`。
2. 或删掉一个暂时不用的组件。
3. 或换成更宽松的模板，比如 `redline_2077 / ember_status`。

编辑器里现在会显示：

- 当前网格总格数
- 已占用格数
- 剩余格数

添加失败时也会提示想放进去的是多大的块。

## 5. 图片怎么加

图片组件现在支持单独配置路径、显示模式和渲染模式。

使用方法：

1. 在编辑器中添加 `ImageWidget`。
2. 选中这个图片组件。
3. 在右侧 `Image Display` 里选择显示方式：
   - `Scale to Fit`：等比缩放，完整显示。
   - `Fill and Crop`：等比填满，必要时会裁边。
   - `Stretch`：强制拉伸到组件尺寸。
4. 在右侧 `Image Render` 里选择渲染方式：
   - `ASCII`：沿用原有字符画方式，更像传统终端。
   - `Pixel`：使用彩色半块字符拼成伪像素图，通常更清晰，也更适合主视觉大图。
5. 在右侧 `Image Path` 设置图片路径。
6. 你可以直接手输，也可以：
   - 点 `Browse...` 打开系统文件选择器。
   - 点 `Paste Path` 从系统剪贴板粘贴路径。
   - 在输入框聚焦时按 `Ctrl+V`，编辑器会优先读取系统剪贴板。
7. 等输入完成并生效后，点 `Save Layout` 保存到布局文件。
8. 返回主界面后会自动读取该路径、显示模式和渲染模式。

路径规则：

- 支持绝对路径。
- 也支持相对路径。
- 相对路径会优先尝试：
  - 当前运行目录
  - 应用数据目录
  - 程序目录

说明：

- 为空时，默认使用内置 `assets/logo.png`。
- 如果路径不存在，组件会显示 `No Image Loaded`。
- 如果旧布局没有配置显示模式，现在默认会按 `Scale to Fit` 处理，避免强制拉伸变形。
- 如果旧布局没有配置渲染模式，现在默认会按 `ASCII` 处理，以保持兼容。

## 6. Visual Variant 是什么

`Visual Variant` 可以理解为“同一个组件的外观版本”，不会改变组件类型本身，但会改变它更像哪种视觉语言。

目前常见的有两类：

- 传统面板类
  - `Dense / Slim / Compact / Minimal / Banner / Glow / Outline`
- 轻框图形类
  - `Rail`：像一条信息轨道。
  - `Corner`：更像角标+标签的轻框块。
  - `Ribbon`：更像横向信号条。
  - `Hero`：弱化边框，突出主体内容，适合图片和时钟这种主视觉组件。

如果你想减少“满屏矩形框”的感觉，可以优先尝试 `Rail / Corner / Ribbon / Hero` 这几种。

## 7. 字体能不能更丰富

可以，而且不一定要内置在项目里。

当前要分成两层看：

- App 内部“字体预设”
  - 实际上是字符风格预设，比如条形字符、趋势线字符、图像字符集。
  - 它已经会跟模板一起切换。
- Windows Terminal 的真实显示字体
  - 这是终端宿主层的字体，比如 `Cascadia Mono`、`Consolas`、`Lucida Console`、`Courier New`。
  - 这里本来就支持填写任意已安装字体名。

配置位置：

- 用户设置文件：`settings.json5`
- 关键字段：`terminal_integration.bundled_wt_font_face`

示例：

```json5
{
  terminal_integration: {
    bundled_wt_font_face: "Consolas",
    bundled_wt_font_size: 15
  }
}
```

建议：

- 优先选等宽字体。
- 改完字体后重新启动程序，效果最稳定。
- 如果用户机器没装这个字体，WT 会回退，不会把程序搞崩。

## 8. 当前已知限制

- Textual 版编辑器目前还是“网格布局 + 数值输入”为主，不是完全可视化拖拽排版器。
- 图片组件现在已支持系统文件选择器与系统剪贴板粘贴，但还不支持把文件直接拖进 Textual 编辑器。
- 主界面底部 Footer 里的 `Zoom -/+` 调的是 Windows Terminal 宿主缩放，和模板里的 `global_scale` 不是同一个概念。
- 当前 `Pixel` 渲染模式已接入静态图片组件，但 GIF 动画还没有并到同一套新渲染链里。
